API Documentation for the Business Connection Platform

This document provides details on the API endpoints available in the system, including request and response formats.

---

## Base URL

The base URL for the API is `/api/`.

---

## Authentication

Endpoints that require authentication expect a JSON Web Token (JWT) in the `Authorization` header:

`Authorization: Bearer <access_token>`

---

## Standard Response Format

The API returns responses in a standardized format:

**Success Response:**
```json
{
    "success": true,
    "message": "A descriptive message",
    "data": { ... }
}
```

**Error Response:**
```json
{
    "success": false,
    "message": "A descriptive error message",
    "errors": { ... }
}
```

## Business Account API (`/api/business-account/`)

### 1. Registration
- **Endpoint:** `signup/`
- **Method:** `POST`

### 2. Login
- **Endpoint:** `login/`
- **Method:** `POST`

### 3. Profile Registration (Complete Profile)
- **Endpoint:** `profile-register/`
- **Method:** `PATCH` / `PUT`
- **Authentication:** Required (Business Account Token)
- **Request Body (JSON):**
```json
{
    "role_position": "CEO",
    "business_name": "Tech Corp",
    "industry_category": "Software",
    "business_email": "contact@techcorp.com",
    "website": "https://techcorp.com",
    "address": "123 Main St",
    "city": "New York",
    "state": "NY",
    "zip_code": "10001"
}
```

### 4. Verification Requests (`/api/business-account/verification/`)

#### Request Verification (User Only)
- **Endpoint:** `request/`
- **Method:** `POST`
- **Authentication:** Required (Standard User Token)
- **Request Body (JSON):**
```json
{
    "business_account": "business-account-uuid"
}
```
- **Description:** Allows a standard user to request profile verification from a specific business.

#### List Pending Verification Requests (Business Only)
- **Endpoint:** `requests/`
- **Method:** `GET`
- **Authentication:** Required (Business Account Token)
- **Description:** Returns a list of all pending verification requests received by the business.

#### Verification Request Action (Business Only)
- **Endpoint:** `requests/<uuid:request_id>/action/`
- **Method:** `POST`
- **Authentication:** Required (Business Account Token)
- **Request Body (JSON):**
```json
{
    "action": "accept" 
}
```
- **Actions:** `accept` or `reject`.
- **Behavior:** Accepting a request marks the user as `is_verified`.

### 5. Membership Management (`/api/business-account/`)

#### List Verified Members (Business Only)
- **Endpoint:** `members/`
- **Method:** `GET`
- **Authentication:** Required (Business Account Token)
- **Description:** Lists all users verified by this business.

#### Remove Verified Member (Business Only)
- **Endpoint:** `members/<uuid:user_id>/remove/`
- **Method:** `POST`
- **Authentication:** Required (Business Account Token)
- **Description:** Removes a user from the verified member list. If the user has no other verifications, their `is_verified` status is revoked.

### 1. Common Post Features

#### Tagging (Social Media Style)
When creating or updating a post, use the `tags` field with a list of strings. Tags are automatically converted to lowercase.
- **Input:** `"tags": ["UIUX", "Backend"]`
- **Output:** Returns `tags_detail` with full tag objects.

#### Images
Images should be uploaded using `multipart/form-data`.
- **Field Name:** `images` (can be multiple files)
- **Optional Captions:** `images_caption` (list of strings corresponding to files)

---

### 2. Need Posts

#### List All Need Posts
- **Endpoint:** `needs/`
- **Method:** `GET`
- **Authentication:** Required
- **Query Params:** `?tag=tagname` (Filter by tag)
- **Description:** Returns a list of all `NeedPosts` in the system.

#### Create a Need Post
- **Endpoint:** `needs/`
- **Method:** `POST`
- **Authentication:** Required
- **Request Body (`POST` - multipart/form-data):**
  - `title`: "Need a UI Designer"
  - `description`: "Looking for an expert UI designer for a 3-month project."
  - `category`: "design" (one of: `technology`, `logistic`, `marketing`, `legal`, `suply_chain`, `finance`, `design`, `other`)
  - `tags`: ["UIUX", "Figma"] (multiple fields or JSON array)
  - `images`: (file)
  - `images_caption`: "Mockup draft"

- **Response Example (`data`):**
```json
{
    "id": "uuid-here",
    "author_id": "author-uuid",
    "author_type": "user",
    "title": "Need a UI Designer",
    "description": "Looking for an expert UI designer for a 3-month project.",
    "category": "design",
    "tags_detail": [
        {"id": 1, "name": "uiux"},
        {"id": 2, "name": "figma"}
    ],
    "images": [
        {"image": "/media/post_images/img.jpg", "caption": "Mockup draft"}
    ],
    "created_at": "2026-03-29T04:00:00Z",
    "updated_at": "2026-03-29T04:00:00Z"
}
```

### 3. Need Post Proposals

#### Submit a Proposal
- **Endpoint:** `needs/<uuid:post_id>/propose/`
- **Method:** `POST`
- **Authentication:** Required
- **Request Body (multipart/form-data):**
  - `subject`: "UI Design Proposal" (Short summary)
  - `message`: "I can help with your UI design project. I have 5 years of experience." (Text field for cover letter)
  - `cv_file`: (File upload) Supporting document, CV (PDF or Image)
- **Constraint:** A user cannot propose to their own post, and can only submit one proposal per post.

#### List Proposals for a Post
- **Endpoint:** `needs/<uuid:post_id>/proposals/`
- **Method:** `GET`
- **Authentication:** Required (Post Author Only)

#### Accept or Reject a Proposal
- **Endpoint:** `proposals/<uuid:proposal_id>/action/`
- **Method:** `POST`
- **Authentication:** Required (Post Author Only)
- **Request Body (JSON):**
```json
{
    "action": "accept"
}
```
- **Actions:** `accept` or `reject`.
- **Description:** Approving a connection request (proposal) allows both parties to start a discussion.

#### List All Received Proposals
- **Endpoint:** `proposals/received/`
- **Method:** `GET`
- **Authentication:** Required.
- **Behavior:** Returns all proposals sent to all posts created by the currently authenticated user.

#### Cancel a Proposal
- **Endpoint:** `proposals/<uuid:pk>/cancel/`
- **Method:** `POST`
- **Authentication:** Required (Proposer Only)
- **Description:** Cancels a submitted proposal or inquiry for both Need and Offer posts.

### 7. Post Detail (Retrieve, Update, Delete)

- **Endpoints:** 
  - `needs/<uuid:pk>/` (Type-specific)
  - `offers/<uuid:pk>/` (Type-specific)
  - `<uuid:pk>/` (Generic - Retrieve only)
- **Methods:** `GET`, `PUT`, `PATCH`, `DELETE`
- **Permissions:** 
  - `GET`: Any authenticated user.
  - `PUT`, `PATCH`, `DELETE`: Only the author.
- **Generic Endpoint Behavior:** The `<uuid:pk>/` endpoint automatically detects if the post is a "Need" or an "Offer" and returns the details using the combined serializer format.

## Chat & Messaging API (`/api/chat/`)

### 0. Real-time WebSockets (WebSocket)
- **Endpoint:** `ws/chat/<uuid:conversation_id>/?token=<access_token>`
- **Protocol:** `ws://` (local) or `wss://` (production)
- **Usage:** Connect to this socket for real-time updates.
- **Message Format (to send):** `{"message": "Hello world!"}`
- **Message Format (received):** `{"message": "...", "sender_id": "...", "sender_type": "...", "created_at": "..."}`

### 1. List Conversations (Inbox)
- **Endpoint:** `conversations/`
- **Method:** `GET`
- **Authentication:** Required
- **Behavior:** Returns all chat threads the user is part of. This includes "Message Requests" (pending connection requests) and "Active Chats".

### 2. Message History & Send Message (REST Backup)
- **Endpoint:** `conversations/<uuid:conversation_id>/messages/`
- **Methods:** `GET`, `POST`
- **Authentication:** Required (Participant Only)
- **POST Body:** `{"text": "Hello, I am interested!"}`
- **Constraint:** You can only send messages in `active` conversations.

### 3. Accept Message Request
- **Endpoint:** `conversations/<uuid:conversation_id>/action/`
- **Method:** `POST`
- **Authentication:** Required (Post Owner Only)
- **POST Body:** `{"action": "accept"}`
- **Behavior:** Transitions the chat to `active`.

### 4. Start General Conversation (Direct Message)
- **Endpoint:** `conversations/start/`
- **Method:** `POST`
- **Authentication:** Required
- **Request Body (JSON):** `{"user_id": "other-user-uuid"}`
- **Behavior:** Starts a 1-to-1 direct chat with any user. These are `active` by default.
